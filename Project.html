Boston Location Prices Model

Coursera Developing Data Product Project

Introduction

In this project, we apply a basic linear model to predict housing prices around Boston. Additionally, leaflet is used to provide a geographic interface to the data.

The dataset can be downloaded from here

(Note: the dataset does not contain any latitude and longitude data. For the purposes of this project, random locations were generated around the boston area.)

The Data

  • RM - average number of rooms among homes in the neighborhood.
  • LSTAT - the percentage of homeowners in the neighborhood considered "lower class" (working poor).
  • PTRATIO - the ratio of students to teachers in primary and secondary schools in the neighborhood.
  • MEDV - the price of a house.
  • lng - longitude.
  • lat - latitude.

Shiny Application

A shiny application is used to allow the user to interact with the data. The application can be found at [https://damienpond.shinyapps.io/Project/]

The application allows interaction on two levels:

  1. A linear model calculates an expected house price based on inputs, which can be adjusted by the user.
  2. An interactive map gives location and heatmap information, as well as calculating the mean, max and min house prices within a given radius.
